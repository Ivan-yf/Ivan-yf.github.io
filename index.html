冒泡排序（Bubble Sort）

冒泡排序是用得比较多而且比较通俗易懂的排序算法，重复地遍历要排序的数组，一次比较两个元素，如果两个元素逆序，则交换，不断重复直到不需要再交换，则完成排序。（时间复杂度O(n²)）

步骤（以下排序都是按从小到大）：

  1. 比较相邻的两个元素，如果是第一个元素比第二个元素大，则交换两个元素的位置

  2. 遍历数组，两两元素重复第一步的操作，当index到最后一个元素时，这步走完，最后一个元素将是此数组中最大的一个元素

  3. 不断重复第一步跟第二步的操作，当然最后那些已排好的元素不用再参与，直到没有一对元素需要比较

    public int[] bubble(int arr[]){
    int temp = 0;
    boolean flag = false;
    for(int i = 0;i < arr.length - 1;i++){
        for(int j = 0;j < arr.length - i - 1;j++){
            if(arr[j] > arr[j+1]){
                temp = arr[j+1];
                arr[j+1] = arr[j];
                arr[j] = temp;
                flag = true;
            }
        }
        if(!flag){
            break;
        }
    }
    return arr;
}

选择排序（Select Sort）

选择排序比较直观，遍历找到一个比N小并且是在N+1到数组最大长度中这个范围内的最小值，与N进行交换，如此循环（时间复杂度O(n²)）

步骤：

  1. 在未排序的数组序列中找到一个最小值，放在排序序列的初始位置

  2. 接着再从未排序序列中找最小值，放在已排序序列的末尾位置

  3. 重复第二步的操作，知道未排序序列中没有值

    public int[] select(int[] nums){
    for(int i = 0;i < nums.length - 1;i++){
        int min = nums[i];
        int index = i;
        for(int j = i + 1;j < nums.length;j++){
            if(nums[j] < min){
                min = nums[j];
                index = j;
            }
        }
        nums[index] = nums[i];
        nums[i] = min;
    }
    return nums;
}

插入排序（Insert Sort）

插入排序类似选择排序，不过没有它粗暴，但是比较直观，同样是分已排序序列跟未排序序列，对于每个未排序序列，从已排序序列的末尾往前，找到它对应的位置插入（时间复杂度O(n²)）

步骤：

  1. 将未排序的第一个元素当成已排好序的序列，把第二个元素开始往后当成未排序序列

  2. 在未排序序列中从第N个元素开始，每次取未排序序列的第一个元素拿到已排好序序列中从后往前比较，如果被比较的元素比当前元素大，则将被比较的元素往后移一位下标，如果当找到前一个元素比它小，后一个元素比它大时，将此元素插入到这个位置中

  3. 循环第二步，直到未排序序列中没有元素

    public int[] insertSort(int[] nums){
    for(int i = 1;i < nums.length;i++){
        int index = i;
        int temp = nums[i];
        for(int j = i;j > 0;j--){
            if(temp < nums[j-1]){
                nums[j] = nums[j-1];
                index--;
            }
        }
        nums[index] = temp;
    }
    return nums;
}

快速排序（Quick Sort）

快速排序采用了分治思想，也就是先分，再合，将数组分成左右两区。快排有两种方法，左右指针法跟挖坑法。都是定义一个基准，将此数组中比基准小的数放左边，比基准大的数放右边，如此一来就分成两个区，再进行递归（时间复杂度O(nlogn)）

  • 左右指针法

    步骤：

    1. 定义一个基准flag，这里将中位数定为flag

    2. 将数组最左边的元素标记为左标记，将最右边的元素标记为右标记

    3. 先从右标记出发，当右标记的值大于flag值时，右标记不断左移，也就是它的下标不断–；直到找到一个小于或者等于flag的

    4. 然后从左边出发，当左标记的值小于flag值时，左标记不断右移，也就是它的下标不断++；直到找到一个大于或者等于flag的

    5. 此时交换左标记跟右标记的元素值

    6. 再进行判断，如果此时左标记或者右标记的值等于flag，一种情况就是有的元素值刚好等于flag，还有一种情况就是防止左标记的下标或者右标记的下标走到flag的位置上时，与之交换发生原地踏步，所以将左标记下标++或者右标记下标–

    7. 再次移动标记，当两个标记重合时则停止

    8. 此时完成了第一次操作，比基准小的都在基准左边，比基准大的都在基准右边

    9. 然后递归左右区的序列

      public int[] quick1(int arr[],int left,int right){
    int flag = arr[(left + right)/2];
    int l = left;
    int r = right;
    while(l < r){
        while(arr[r] > flag){
            r--;
        }
        while(arr[l] < flag){
            l++;
        }
        if(l >= r){
            break;
        }
        int temp = arr[r];
        arr[r] = arr[l];
        arr[l] = temp;
        if(arr[r] == flag && l < r){
            l++;
        }
        if(arr[l] == flag && l < r){
            r--;
        }
    }
    if(l == r){
        l++;
        r--;
    }
    if(left < r){
        quick1(arr,left,r);
    }
    if(right > l){
        quick1(arr,l, right);
    }
    return arr;
}

  • 挖坑法

    步骤：

    1. 定义一个基准flag，这里将数组第一个元素定为flag

    2. 将数组最左边的元素标记为左标记，将最右边的元素标记为右标记

    3. 先从右标记出发，当右标记的值大于或等于flag值时，右标记不断左移，也就是它的下标不断–；若当右标记值小于flag而且左标记下标还是小于右标记下标，则将此时的右标记的的值赋给左标记，也给此时的右标记下标挖个坑

    4. 然后开始从左标记出发，若左标记的值小于或等于flag值，左标记不断右移，也就是它的下标不断++；，若当左标记的值大于flag而且左标记下标还是小于右标记下标，则将上次挖的坑给填上，也就是将此时的左标记的值赋给上次挖的坑右标记下标值，同时再给此时的左标记下标挖个坑

    5. 如此循环挖坑，当左标记下标跟右标记下标重合或者比它大时，也就是跳出while循环时，记得将最后的坑给填上，将flag丢给它，此时flag左边都小于flag，右边都大于flag

    6. 然后递归左右区的序列

      public int[] quick2(int arr[],int left,int right){
    int flag = arr[left];
    int l = left;
    int r = right;
    while(l < r){
        while(l < r && arr[r] >= flag){
            r--;
        }
        arr[l] = arr[r];
        while(l < r && arr[l] <= flag){
            l++;
        }
        arr[r] = arr[l];
    }
    arr[r] = flag;
    if(l == r){
        l++;
        r--;
    }
    if(left < r){
        quick1(arr,left,r);
    }
    if(right > l){
        quick1(arr,l, right);
    }
    return arr;
}

希尔排序（Shell Sort）

希尔排序是对插入排序的一种改进，以往插入排序将未排序的数组插入已排序数组中时，每次只能移动一位元素，使得效率非常低。希尔排序会优先比较距离较远的元素，先将整个待排序的记录序列分割成为若干子序列分别进行直接插入排序，待整个序列中的记录“基本有序”时，再对全体记录进行依次直接插入排序（时间复杂度O(nlogn)）

步骤：

  1. 选择一个增量，一般是数组长度除2
  2. 根据增量的大小，进行增量次数的排序
  3. 每次排序，将最近增量相隔的两个数进行比较，如逆序则交换
  4. 当排序增量次数的排序结束，再次将增量/2，直到增量值等于一

public int[] shell(int nums[]){
    int number = nums.length/2;
    while(number >= 1){
        for(int i = number;i < nums.length;i++){
            int temp = nums[i];
            int j = i - number;
            while(j >= 0 && temp < nums[j]){
                nums[j + number] = nums[j];
                j -= number;
            }
            nums[j + number] = temp;
        }
        number /= 2;
    }
    return nums;
}

计数排序（Count Sort）

计数排序的核心在于将输入的数据值转化为键存储在额外开辟的数组空间中。作为一种线性时间复杂度的排序，计数排序要求输入的数据必须是有确定范围的整数。同时这也就成了它的缺点，但如果数组的值之间范围相差太大，新开辟的空间就要非常大，比如，一跟一亿，这就相当于要new一个大小为一亿的数组。（时间复杂度O(n+k)）

步骤：

  1. 遍历初始数组，找出最小值min跟最大值max

  2. 生成一个新数组，长度大小为(max - min + 1),新数组的下标对应的值是旧数组对应值的出现个数，这里不好理解，比如旧数组中最大值是10，最小值是2,然后3出现了两次，所以对应的新数组，下标为1的值等于2，也就是两次，下标为0的值代表着最小值2出现的次数；

  3. 最后遍历新数组，根据新数组中各个下标值去赋给旧数组，当然要记得赋给旧数组的时候将新数组的下标对应的值加上min，比如新数组的下标1的值为2，那么要赋给旧数组的话，就相当于3要出现两次。

  4. （补充：这里可以不找出min，就直接找max元素，新数组的大小就等于（max+1），但是生成的新数组就比较耗空间，相反而来，过程就不会那么饶，一会减一会加了）

    public int[] count(int arr[]){
    int min = getMin(arr);
    int max = getMax(arr);
    int[] result = new int[max - min + 1];
    for(int i : arr){
        result[i - min]++;
    }
    int index = 0;
    for(int i = 0;i < result.length;i++){
        while(result[i] > 0){
            arr[index] = i + min;
            result[i]--;
            index++;
        }
    }
    return arr;
}
private int getMin(int arr[]){
    int temp = arr[0];
    for(int i : arr){
        if(i < temp){
            temp = i;
        }
    }
    return temp;
}
private int getMax(int arr[]){
    int temp = arr[0];
    for(int i : arr){
        if(i > temp){
            temp = i;
        }
    }
    return temp;
}

归并排序（Merge Sort）

这里同样采用了分治思想，想将原本数组一次又一次地对半拆分，直到每片区域只有一个元素，就算拆分完成，然后将拆分的每片区域进行合并，合并时，按升序比较，将较小的值先丢进申请的数组空间中，按同样的逻辑重复比较，直到所有的值都在一个区域，也就是同个数组

步骤：

  1. 首先将原本数组不断拆分成左右两片区域，知道区域中元素个数为1
  2. 设定两个指针，最初位置分别为两个已经排序序列的起始位置
  3. 比较拆分后的两个区域的元素，也就是比较两指针指向的元素，将相对小的元素放入到合并的空间中，并将指针移动到下一位
  4. 循环第三步直到左右某一区域到达末尾
  5. 将未到达末尾的区域剩余部分全部加入到合并区域中

public void merge(int[] arr,int left,int right,int[] temp){
    if(left < right){
        int mid = (left + right) / 2;
        merge(arr,left,mid,temp);
        merge(arr,mid+1,right,temp);
        mergeResult(arr,left,mid,right,temp);
    }
}
public void mergeResult(int arr[],int left,int mid,int right,int[] temp){
    int k = 0;
    int i = left;
    int j = mid + 1;
    while(i <= mid && j <= right){
        if(arr[i] <= arr[j]){
            temp[k] = arr[i];
            i++;
            k++;
        }else{
            temp[k] = arr[j];
            j++;
            k++;
        }
    }
    while(i <= mid){
        temp[k] = arr[i];
        i++;
        k++;
    }
    while(j <= right){
        temp[k] = arr[j];
        j++;
        k++;
    }
    k = 0;
    int tempLeft = left;
    while(tempLeft <= right){
        arr[tempLeft] = temp[k];
        tempLeft++;
        k++;
    }
}

基数排序（Radix Sort）

基于桶排思想，生成十个桶，对应0-9，按照数组中最大的那个数的位数进行循环，从个位数开始取模，得到的值插入相对应的index桶中，并将此序列桶的元素个数标记加一，最后再将桶中的元素重新赋给原来的数组，再进行循环。（时间复杂度O(n*k)）

步骤：

  1. 循环初始数组，找出最大值，并找出最大值的长度

  2. 定义一个二维数组，也就是桶，第一个下标代表桶的顺序，第二个代表桶中的值，（这里用了数组来暂放元素，由于空间不确定，所以生成了一个很大数组，当然也可以用链表的方式。）再定一个一维数组，表示桶中各元素的位置。

  3. 从低位开始，计算初始数组中各个元素的低位数的值，并放入相对应的桶中

  4. 如此一次后，将各个不为空的桶的值按照桶的顺序一一取出来。

  5. 再进行高位的计算，循环34步骤，直到最高位排序完成

    public int[] radix(int[] arr){
    int [][]bucket = new int[10][arr.length];
    int []bucketIndex = new int[10];
    int max = arr[0];
    for(int i = 1;i < arr.length;i++){
        if(arr[i] > max){
            max = arr[i];
        }
    }
    int length = (max + "").length();
    for(int j = 0,n = 1;j < length;j++,n *= 10){
        for(int i = 0;i < arr.length;i++){
            int digit = arr[i] / n % 10;
            bucket[digit][bucketIndex[digit]] = arr[i];
            bucketIndex[digit]++;
        }
        int k = 0;
        for(int i = 0;i < bucketIndex.length;i++){
            if(bucketIndex[i] != 0){
                for(int l = 0;l < bucketIndex[i];l++){
                    arr[k++] = bucket[i][l];
                }
            }
            bucketIndex[i] = 0;
        }
    }
    return arr;
}

1.缓存雪崩：

当redis服务器重启或者当大量key需要同时更新或者失效时，此时大量的请求就会请求缓存，访问不到只能打到数据库上，造成数据库宕机.

解决办法：

1）均匀设置key的失效时间，使其不要在同一时间内大量失效
2）加锁排队
3）一般发生大量访问的原因都是做活动，所以可以先预热缓存，先更新缓存

2.缓存击穿：
指一个非常热点的key，不停的被访问，也就是高并发，集中对一个点进行访问，同样当这个key失效的瞬间，大量的高并发就穿透缓存，直接打到数据库上。

3.缓存穿透：

用户想要查询一个数据，发现redis缓存中没有，也就是缓存没有命中，所以缓存中访问不到，一般我们都会去数据库中查出来然后加到缓存中。
但是发现数据库中也没有这个数据，于是查询失败。
当有用户抓住这个漏洞，持续访问这个不存在的key，所以一直在访问持久层数据库，给之造成了很大压力，这个时候就出现了缓存穿透。

解决办法：

1）缓存空对象：当数据库查不到此数据时，返回一个空对象，缓存将这个空对象也缓存起来，并设置一个过期时间，之后用户再访问时就会直接在缓存中访问到。
2）布隆过滤器（以下为百度的东西）

布隆过滤器的应用：
我们先来看下布隆过滤器的应用场景，让大家知道神奇的布隆过滤器到底能做什么。

现在有大量的数据，而这些数据的大小已经远远超出了服务器的内存，现在再给你一个数据，如何判断给你的数据在不在其中。
如果服务器的内存足够大，那么用HashMap是一个不错的解决方案，理论上的时间复杂度可以达到O(1)，
但是现在数据的大小已经远远超出了服务器的内存，所以无法使用HashMap，这个时候就可以使用“布隆过滤器”来解决这个问题。
但是还是同样的，会有一定的“误判率”。

什么是布隆过滤器：
布隆过滤器是一个叫“布隆”的人提出的，它本身是一个很长的二进制向量，既然是二进制的向量，那么显而易见的，存放的不是0，就是1。

可以看出，仅仅从布隆过滤器本身而言，根本没有存放完整的数据，只是运用一系列随机映射函数计算出位置，然后填充二进制向量。

这有什么用呢？比如现在再给你一个数据，你要判断这个数据是否重复，你怎么做？

你只需利用上面的三种固定的计算方式，计算出这个数据占据哪些格子，然后看看这些格子里面放置的是否都是1，
如果有一个格子不为1，那么就代表这个数字不在其中。这很好理解吧，比如现在又给你了刚才你添加进去的数据，
你通过三种固定的计算方式，算出的结果肯定和上面的是一模一样的，也是占据了布隆过滤器“5”，“9”，“2”三个格子。

但是有一个问题需要注意，如果这些格子里面放置的都是1，不一定代表给定的数据一定重复，也许其他数据经过三种固定的计算方式算出来的结果也是相同的。这也很好理解吧，比如我们需要判断对象是否相等，是不可以仅仅判断他们的哈希值是否相等的。

也就是说布隆过滤器只能判断数据是否一定不存在，而无法判断数据是否一定存在。

按理来说，介绍完了新增、查询的流程，就要介绍删除的流程了，但是很遗憾的是布隆过滤器是很难做到删除数据的，为什么？你想想，比如你要删除刚才给你的数据，你把“5”，“9”，“2”三个格子都改成了0，但是可能其他的数据也映射到了“5”，“9”，“2”三个格子啊，这不就乱套了吗？

相信经过我这么一介绍，大家对布隆过滤器应该有一个浅显的认识了，至少你应该清楚布隆过滤器的优缺点了：

优点：由于存放的不是完整的数据，所以占用的内存很少，而且新增，查询速度够快；
缺点： 随着数据的增加，误判率随之增加；无法做到删除数据；只能判断数据是否一定不存在，而无法判断数据是否一定存在。
可以看到，布隆过滤器的优点和缺点一样明显。

在上文中，我举的例子二进制向量长度为16，由三个随机映射函数计算位置，在实际开发中，如果你要添加大量的数据，仅仅16位是远远不够的，为了让误判率降低，我们还可以用更多的随机映射函数、更长的二进制向量去计算位置。

八皇后问题(递归回溯实现)

2020-02-24

wiki：八皇后问题，是一个古老而著名的问题，是回溯算法的典型案例。该问题是国际西洋棋棋手马克斯·贝瑟尔于1848年提出：在8×8格的国际象棋上摆放八个皇后，使其不能互相攻击，即任意两个皇后都不能处于同一行、同一列或同一斜线上，问有多少种摆法。

摆法（暴力）：

  1. 第一个皇后先放在第一行第一列
  2. 第二个皇后放在第二行第一列，然后判断是否跟前面的摆法的皇后冲突，如果冲突，则继续往第二列，第三列摆法，直到找到一个合适的或者达到最后一列
  3. 继续第三个皇后，还是第一列，第二列，直到第八个皇后放在不冲突的位置，就算找到了一个正确解
  4. 当得到一个正确解时，在栈回退到上一个栈时，就会开始回溯，即将第一个皇后放在第一列的所有正确解全部得到
  5. 然后回头继续讲第一个皇后放到第二列，继续循环求解

这里可以采用一维数组即可，queen[8],对应的queen下标代表第几个皇后，对应下标的值表示第几个皇后在第几列。而不用通过说利用二维数组表示8*8棋盘的位置，这样会浪费数组空间。

package recall;

/**
 * 概要
 *
 * @Author Ivan
 * @vesion 1.0
 * @since 0.1.0
 */
public class Queen8 {
    public static void main(String[] args) {
        Queen8 queen = new Queen8();
        queen.way(0);
        System.out.println("总共有" + count + "种解法");
    }
    int queen[] = new int[8];
    static int count = 0;

    /*
     *  每一次递归时，都有一套for循环，当子函数执行完后，会回溯到上一层的for循环中（有点难理解）
     *
     *  @Param [n]
     *  @Return void
     *  @Author Ivan
     */
    private void way(int n){
        if(n == 8){//当达到第九（n=8）个皇后时，即表示已找到正确解，输出并打印，然后回溯到上一层
            print();
            return;
        }
        for(int i = 0;i < queen.length;i++){//每一个皇后都要进行for循环遍历完8列去找合适的位置
            queen[n] = i;//将当前的皇后放在第i个位置
            //判断当前皇后放到i位置时，是否会发生冲突
            if(conflict(n)){
                //如果不冲突，则开始递归，放入第n+1个皇后
                way(n + 1);
            }
            //如果冲突，则接着for循环，将当前皇后的位置放到i+1列的位置
        }
    }
    /*
     * 判断是否当前皇后是否与前面所有已摆放的皇后位置上有冲突
     *
     *  @Param [n]
     *  @Return boolean
     *  @Author Ivan
     */
    private  boolean conflict(int n){
        for(int i = 0;i < n;i++){
            //冲突的条件：与其他皇后在同一列，或者同一斜线
            if(queen[i] == queen[n] || Math.abs(n - i) == Math.abs(queen[n] - queen[i])){
                return false;
            }
        }
        return true;
    }
    private  void print(){
        for(int i = 0;i < queen.length;i++){
            System.out.print(queen[i] + " ");
        }
        count++;
        System.out.println();
    }
}

约瑟夫环

2020-02-17

约瑟夫（Josephu）环

百度：约瑟夫问题（有时也称为约瑟夫斯置换，是一个出现在计算机科学和数学中的问题。在计算机编程的算法中，类似问题又称为约瑟夫环。又称“丢手绢问题”.）

构建一个单项的环形链表

定义一个节点类

class Boy{
    private int no;//编号
    private Boy next;//指向下一个节点，默认null
    Boy(int no){
        this.no = no;
    }
    public int getNo() {
        return no;
    }

    public void setNo(int no) {
        this.no = no;
    }

    public Boy getNext() {
        return next;
    }

    public void setNext(Boy next) {
        this.next = next;
    }
}
  1. 先创建第一个节点，让first指向该节点，并形成环形；
  2. 后面当我们每创建一个新的节点，就把该节点加入到已有的环形链表中即可.
class CircleLinkList{
//创建一个first节点
    private Boy first = null;
    //添加小孩节点，构建环形的链表
    public void addBoy(int nums){
        if(nums < 1){
            System.out.println("nums的值不正确");
            return;
        }
        Boy curBoy = null;//辅助指针，帮助构建环形链表
        for(int i = 1;i <= nums;i++){
            //根据编号创建小孩节点
            Boy boy = new Boy(i);
            if(i == 1){
                first = boy;
                first.setNext(first) ;//构成环
                curBoy = first;//让curBoy指向第一个节点
            }
            else{
                curBoy.setNext(boy);//将上一个节点与当前新生成的节点相连
                boy.setNext(first);//将当前新生成的节点与头结点相连形成环
                curBoy = boy;//将curBoy节点指向新生的节点
            }
        }
    }
}

遍历环形链表

  1. 先让一个辅助指针（变量）curBoy，指向first节点；
  2. 然后通过一个while循环遍历该环形链表，当curBoy.next==first结束
class CircleLinkList{
//创建一个first节点
    private Boy first = null;
    //遍历环形链表
    public void show(){
        if(first == null){
            System.out.println("链表为空");
            return;
        }
        Boy curBoy = first;
        while(true){
            System.out.println(curBoy.getNo());
            //判断是否到最后
            if(curBoy.getNext() == first){
                break;
            }
            curBoy = curBoy.getNext();
        }
    }
}

环形出圈

根据用户的输入，生成一个小孩出圈的顺序

n=5，即有5个人

k=1，从第一个人开始报数

m=2，数2下然后出圈

  1. 需求创建一个辅助指针（变量）helper，事先应该指向环形链表的最后这个节点小孩报数前，先让first跟helper移动k-1次
  2. 当小孩报数时，让first和helper指针同时移动m-1次
  3. 这时就可以让first指向的小孩节点出圈

first = first.next

helper.next = first

原来first指向的节点就没有任何引用==出圈回收

class CircleLinkList{
    //创建一个first节点
    private Boy first = null;

    //根据用户的输入，计算出圈的顺序
    /**
     *
     * @param startNo 初始在第几个小孩开始数数
     * @param countNo 数多少下出圈
     * @param nums 总共多少个小孩
     */
    public void countBoy(int startNo,int countNo,int nums){
        if(first == null || startNo < 1 || startNo > nums || countNo < 1){
            System.out.println("参数有误");
            return;
        }
        Boy helper = first;
        //将辅助指针指向最后一个节点
        while(true){
            if(helper.getNext() == first){
                break;
            }
            helper = helper.getNext();
        }
        //报数前先让两个节点移动到要报数的位置
        for(int i = 1;i < startNo;i++){
            helper = helper.getNext();
            first = first.getNext();
        }
        //让first和helper指针同时移动countNo-1次,然后出圈
        while (true){
            if(helper == first){
                break;
            }
            for(int i = 0;i < countNo - 1;i++){
                first = first.getNext();
                helper = helper.getNext();
            }
            System.out.println("小孩"+first.getNo()+"出圈");
            first = first.getNext();//将first指向出圈小孩的下一个节点
            helper.setNext(first);//将helper的与出圈小孩的下一个节点相连，重新形成环形
        }
        System.out.println("最后留在圈中的是"+first.getNo());
    }
}
