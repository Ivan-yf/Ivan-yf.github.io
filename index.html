redis

发表于 2020-04-11

1.缓存雪崩：

当redis服务器重启或者当大量key需要同时更新或者失效时，此时大量的请求就会请求缓存，访问不到只能打到数据库上，造成数据库宕机.

解决办法：

1）均匀设置key的失效时间，使其不要在同一时间内大量失效
2）加锁排队
3）一般发生大量访问的原因都是做活动，所以可以先预热缓存，先更新缓存

2.缓存击穿：
指一个非常热点的key，不停的被访问，也就是高并发，集中对一个点进行访问，同样当这个key失效的瞬间，大量的高并发就穿透缓存，直接打到数据库上。

3.缓存穿透：

用户想要查询一个数据，发现redis缓存中没有，也就是缓存没有命中，所以缓存中访问不到，一般我们都会去数据库中查出来然后加到缓存中。
但是发现数据库中也没有这个数据，于是查询失败。
当有用户抓住这个漏洞，持续访问这个不存在的key，所以一直在访问持久层数据库，给之造成了很大压力，这个时候就出现了缓存穿透。

解决办法：

1）缓存空对象：当数据库查不到此数据时，返回一个空对象，缓存将这个空对象也缓存起来，并设置一个过期时间，之后用户再访问时就会直接在缓存中访问到。
2）布隆过滤器（以下为百度的东西）

布隆过滤器的应用：
我们先来看下布隆过滤器的应用场景，让大家知道神奇的布隆过滤器到底能做什么。

现在有大量的数据，而这些数据的大小已经远远超出了服务器的内存，现在再给你一个数据，如何判断给你的数据在不在其中。
如果服务器的内存足够大，那么用HashMap是一个不错的解决方案，理论上的时间复杂度可以达到O(1)，
但是现在数据的大小已经远远超出了服务器的内存，所以无法使用HashMap，这个时候就可以使用“布隆过滤器”来解决这个问题。
但是还是同样的，会有一定的“误判率”。

什么是布隆过滤器：
布隆过滤器是一个叫“布隆”的人提出的，它本身是一个很长的二进制向量，既然是二进制的向量，那么显而易见的，存放的不是0，就是1。

可以看出，仅仅从布隆过滤器本身而言，根本没有存放完整的数据，只是运用一系列随机映射函数计算出位置，然后填充二进制向量。

这有什么用呢？比如现在再给你一个数据，你要判断这个数据是否重复，你怎么做？

你只需利用上面的三种固定的计算方式，计算出这个数据占据哪些格子，然后看看这些格子里面放置的是否都是1，
如果有一个格子不为1，那么就代表这个数字不在其中。这很好理解吧，比如现在又给你了刚才你添加进去的数据，
你通过三种固定的计算方式，算出的结果肯定和上面的是一模一样的，也是占据了布隆过滤器“5”，“9”，“2”三个格子。

但是有一个问题需要注意，如果这些格子里面放置的都是1，不一定代表给定的数据一定重复，也许其他数据经过三种固定的计算方式算出来的结果也是相同的。这也很好理解吧，比如我们需要判断对象是否相等，是不可以仅仅判断他们的哈希值是否相等的。

也就是说布隆过滤器只能判断数据是否一定不存在，而无法判断数据是否一定存在。

按理来说，介绍完了新增、查询的流程，就要介绍删除的流程了，但是很遗憾的是布隆过滤器是很难做到删除数据的，为什么？你想想，比如你要删除刚才给你的数据，你把“5”，“9”，“2”三个格子都改成了0，但是可能其他的数据也映射到了“5”，“9”，“2”三个格子啊，这不就乱套了吗？

相信经过我这么一介绍，大家对布隆过滤器应该有一个浅显的认识了，至少你应该清楚布隆过滤器的优缺点了：

优点：由于存放的不是完整的数据，所以占用的内存很少，而且新增，查询速度够快；
缺点： 随着数据的增加，误判率随之增加；无法做到删除数据；只能判断数据是否一定不存在，而无法判断数据是否一定存在。
可以看到，布隆过滤器的优点和缺点一样明显。

在上文中，我举的例子二进制向量长度为16，由三个随机映射函数计算位置，在实际开发中，如果你要添加大量的数据，仅仅16位是远远不够的，为了让误判率降低，我们还可以用更多的随机映射函数、更长的二进制向量去计算位置。

八皇后问题(递归回溯实现)

发表于 2020-02-24

wiki：八皇后问题，是一个古老而著名的问题，是回溯算法的典型案例。该问题是国际西洋棋棋手马克斯·贝瑟尔于1848年提出：在8×8格的国际象棋上摆放八个皇后，使其不能互相攻击，即任意两个皇后都不能处于同一行、同一列或同一斜线上，问有多少种摆法。

摆法（暴力）：

  1. 第一个皇后先放在第一行第一列
  2. 第二个皇后放在第二行第一列，然后判断是否跟前面的摆法的皇后冲突，如果冲突，则继续往第二列，第三列摆法，直到找到一个合适的或者达到最后一列
  3. 继续第三个皇后，还是第一列，第二列，直到第八个皇后放在不冲突的位置，就算找到了一个正确解
  4. 当得到一个正确解时，在栈回退到上一个栈时，就会开始回溯，即将第一个皇后放在第一列的所有正确解全部得到
  5. 然后回头继续讲第一个皇后放到第二列，继续循环求解

这里可以采用一维数组即可，queen[8],对应的queen下标代表第几个皇后，对应下标的值表示第几个皇后在第几列。而不用通过说利用二维数组表示8*8棋盘的位置，这样会浪费数组空间。

package recall;

/**
 * 概要
 *
 * @Author Ivan
 * @vesion 1.0
 * @since 0.1.0
 */
public class Queen8 {
    public static void main(String[] args) {
        Queen8 queen = new Queen8();
        queen.way(0);
        System.out.println("总共有" + count + "种解法");
    }
    int queen[] = new int[8];
    static int count = 0;

    /*
     *  每一次递归时，都有一套for循环，当子函数执行完后，会回溯到上一层的for循环中（有点难理解）
     *
     *  @Param [n]
     *  @Return void
     *  @Author Ivan
     */
    private void way(int n){
        if(n == 8){//当达到第九（n=8）个皇后时，即表示已找到正确解，输出并打印，然后回溯到上一层
            print();
            return;
        }
        for(int i = 0;i < queen.length;i++){//每一个皇后都要进行for循环遍历完8列去找合适的位置
            queen[n] = i;//将当前的皇后放在第i个位置
            //判断当前皇后放到i位置时，是否会发生冲突
            if(conflict(n)){
                //如果不冲突，则开始递归，放入第n+1个皇后
                way(n + 1);
            }
            //如果冲突，则接着for循环，将当前皇后的位置放到i+1列的位置
        }
    }
    /*
     * 判断是否当前皇后是否与前面所有已摆放的皇后位置上有冲突
     *
     *  @Param [n]
     *  @Return boolean
     *  @Author Ivan
     */
    private  boolean conflict(int n){
        for(int i = 0;i < n;i++){
            //冲突的条件：与其他皇后在同一列，或者同一斜线
            if(queen[i] == queen[n] || Math.abs(n - i) == Math.abs(queen[n] - queen[i])){
                return false;
            }
        }
        return true;
    }
    private  void print(){
        for(int i = 0;i < queen.length;i++){
            System.out.print(queen[i] + " ");
        }
        count++;
        System.out.println();
    }
}

约瑟夫环

发表于 2020-02-17

约瑟夫（Josephu）环

百度：约瑟夫问题（有时也称为约瑟夫斯置换，是一个出现在计算机科学和数学中的问题。在计算机编程的算法中，类似问题又称为约瑟夫环。又称“丢手绢问题”.）

构建一个单项的环形链表

定义一个节点类

class Boy{
    private int no;//编号
    private Boy next;//指向下一个节点，默认null
    Boy(int no){
        this.no = no;
    }
    public int getNo() {
        return no;
    }

    public void setNo(int no) {
        this.no = no;
    }

    public Boy getNext() {
        return next;
    }

    public void setNext(Boy next) {
        this.next = next;
    }
}
  1. 先创建第一个节点，让first指向该节点，并形成环形；
  2. 后面当我们每创建一个新的节点，就把该节点加入到已有的环形链表中即可.
class CircleLinkList{
//创建一个first节点
    private Boy first = null;
    //添加小孩节点，构建环形的链表
    public void addBoy(int nums){
        if(nums < 1){
            System.out.println("nums的值不正确");
            return;
        }
        Boy curBoy = null;//辅助指针，帮助构建环形链表
        for(int i = 1;i <= nums;i++){
            //根据编号创建小孩节点
            Boy boy = new Boy(i);
            if(i == 1){
                first = boy;
                first.setNext(first) ;//构成环
                curBoy = first;//让curBoy指向第一个节点
            }
            else{
                curBoy.setNext(boy);//将上一个节点与当前新生成的节点相连
                boy.setNext(first);//将当前新生成的节点与头结点相连形成环
                curBoy = boy;//将curBoy节点指向新生的节点
            }
        }
    }
}

遍历环形链表

  1. 先让一个辅助指针（变量）curBoy，指向first节点；
  2. 然后通过一个while循环遍历该环形链表，当curBoy.next==first结束
class CircleLinkList{
//创建一个first节点
    private Boy first = null;
    //遍历环形链表
    public void show(){
        if(first == null){
            System.out.println("链表为空");
            return;
        }
        Boy curBoy = first;
        while(true){
            System.out.println(curBoy.getNo());
            //判断是否到最后
            if(curBoy.getNext() == first){
                break;
            }
            curBoy = curBoy.getNext();
        }
    }
}

环形出圈

根据用户的输入，生成一个小孩出圈的顺序

n=5，即有5个人

k=1，从第一个人开始报数

m=2，数2下然后出圈

  1. 需求创建一个辅助指针（变量）helper，事先应该指向环形链表的最后这个节点小孩报数前，先让first跟helper移动k-1次
  2. 当小孩报数时，让first和helper指针同时移动m-1次
  3. 这时就可以让first指向的小孩节点出圈

first = first.next

helper.next = first

原来first指向的节点就没有任何引用==出圈回收

class CircleLinkList{
    //创建一个first节点
    private Boy first = null;

    //根据用户的输入，计算出圈的顺序
    /**
     *
     * @param startNo 初始在第几个小孩开始数数
     * @param countNo 数多少下出圈
     * @param nums 总共多少个小孩
     */
    public void countBoy(int startNo,int countNo,int nums){
        if(first == null || startNo < 1 || startNo > nums || countNo < 1){
            System.out.println("参数有误");
            return;
        }
        Boy helper = first;
        //将辅助指针指向最后一个节点
        while(true){
            if(helper.getNext() == first){
                break;
            }
            helper = helper.getNext();
        }
        //报数前先让两个节点移动到要报数的位置
        for(int i = 1;i < startNo;i++){
            helper = helper.getNext();
            first = first.getNext();
        }
        //让first和helper指针同时移动countNo-1次,然后出圈
        while (true){
            if(helper == first){
                break;
            }
            for(int i = 0;i < countNo - 1;i++){
                first = first.getNext();
                helper = helper.getNext();
            }
            System.out.println("小孩"+first.getNo()+"出圈");
            first = first.getNext();//将first指向出圈小孩的下一个节点
            helper.setNext(first);//将helper的与出圈小孩的下一个节点相连，重新形成环形
        }
        System.out.println("最后留在圈中的是"+first.getNo());
    }
}
